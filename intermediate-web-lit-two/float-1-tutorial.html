Step-by-Step Instructions

  1. What's float?

    TL;DR: The float attribute makes an element stick or wrap to the side of a webpage or container.

    According to the Mozilla Developer Network (MDN), float

    specifies that an element should be taken from the normal flow and placed along the left or right side of its container, where text and inline elements will wrap around it.

    What does that mean?

    When you assign an element, like a div or p (for paragraph) a float value, you tell it to stick as closely as possible to one side or the other of its container - the part of the webpage that holds the element.

    For example, if we float a paragraph with a value of right in our stylesheet (like this: float:right;), then that paragraph will stick to the right side of the webpage's body element or any other container surrounding the paragraph - like header or a div.

    If we have two or more elements floating to the same side of page or container, they will stack horizontally like blocks in a row or beads on an abacus all pushed to one side. We can think also of compartmental eaters here - mashed potatoes float to the left side of the plate; green beans float to the right.

    The float attrbiute takes an element out of its normal space on a page or within a container and shifts towards one side or another.

    While there are other values for float, such as none, left and right are commonly used.

    The float attribute is useful because it lets you create a layout that can expand from something very vertical (on a mobile device) to something very horizontal (on a computer or tablet screen). As the size of your display expands, elements can spread out for readability. As the size of your display contracts, elements with float can still stack vertically (as elements usually do on mobile devices) when they can't wrap horizontally. This way you can have a horizontal layout for computer displays without using positioning that cuts off or loses content on mobile devices. This kind of intentionally scalable layout is called responsive design. We'll do more work with responsive design in another activity.

    Back to navigation.

  2. What's clear?

    TL;DR: The clear attribute breaks the way elements wrap with float.

    Again, according to the MDN, clear

    specifies whether an element can be next to floating elements that precede it or must be moved down (cleared) below them. The clear property applies to both floating and non-floating elements.

    What does that mean?

    When you give an element the clear attribute and a value of left or right, you tell it to create a line-break between itself and whatever floating element comes before it. A cleared element no longer wraps around the floated element that comes before it.

    For example, if I wanted to float two paragraphs or images to the right side of the body of my webpage, but I wanted them both to be on their own lines, I would style both images with float:right;, but also style the second image with clear:right; to float it beneath the image that comes before it. Otherwise, both images would stack horozontally toward the right.

    You can style an element with clear:both; if you want it on its own line and it's between two elements that float.

    Back to navigation.

  3. Rearrange the div elements

    TL;DR: Webotron's parts are in the wrong order to float and clear.

    Webotron's parts are out of order. Before we can use float and clear to rebuild the friendly robot, we need to rearrange the div elements in index.html so that Webotron will stack correctly later in the activity.

    Check out the div elements that hold Webotron's parts beginning at line 38 of index.html (which you can find in th left FILES sidebar). Think about how you would need to order those parts to assemble Webotron correctly with the robot's head on one 'line,' its arms and body on another, and its legs below that.

    Use some notecards, Post-Its, or torn up pieces of paper to help you puzzle out the right layout for our robot.

    Finally, cut and paste the div elements into the right order, but don't stress out over this step too much. You can change tbe order of the parts later on, as well, once you test float and clear.

    Back to navigation.

  4. Play with the float attribute

    TL;DR: See what happens when you change float values.

    After you have reordered the div elements on index.html, go back to the FILES sidebar and go to the file called style.sss. This is the stylesheet that tells the different parts of our webpage how to look and where to be.

    Head down to line 53 where you can find the styling instructions for the div elements that hold Webotron's parts. You can see that right now each part has been assigned a float:left;.

    Play around with the layout of these parts by changing float:left; to float:right;. See what happens when you delete the float lines entirely. Try to explain in your own words what's happening as you change the float value for each piece of Webotron.

    After you mess around with float for a bit, make sure that you add float:left; back to each of Webotron's 'div elements.

    Back to navigation.

  5. Add the clear attribute

    TL;DR: Puzzle out where Webotron's linebreaks belong.

    It's time to rebuild Webotron using clear. You need to create two line breaks using clear:left; to prevent Webotron's body and legs from wrapping to the robot's head.

    Where do you think they should go?

    Add clear:left; to the styling instructions for Webotron's div elements beginning, again, at line 58 of style.css. Experiment until you find the best two spots to add a clear value. Use notecards, Post-Its, or torn up pieces of paper to help visualize what to do.

    When you finish, Webotron should be reassembled and stand tall to represent his band, the Scrambled Divs!

    Back to navigation.
