CSS Building Blocks Tutorial

This project will help you learn to display, position, and style content on a webpage. You will

  1. Know and understand how to use the <div> tag.
  2. Change the display type of blocks made using the <div> tag.
  3. Change the color of those blocks.
  4. Change the size of those blocks.
  5. Change the position of those blocks.

It's a good idea to chunk this tutorial for yourself or your learners. Never be afraid to use the parts that make sense to you and to skip the rest. You can also remix this whole project and delete the parts of the tutorial you don't want to teach right now. If you're not quite ready, for example, to tackle display or positioning, then just start with color and size. If you don't want to worry about picking between websafe color names, hexcodes, and rgba codes in the color section, just pick websafe color names or hexcodes to work on for now. This tutorial is meant to help you, not frustrate you, so don't feel you have to do everything at once!

Step-by-Step Instructions

  1. What's a div?

    TL;DR: a div is an element you can use to create part of a webpage with special rules.

    According to the Mozilla Developer Network, a div is

    the generic container for flow content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang. It should be used only when no other semantic element (such as <article> or <nav>) is appropriate.

    What does that mean?

    Basically, it means that when you create a div and give it an id or class (a sepcial label), you can write instructions on a cascading style sheet (CSS) that create special rules for how that div displays on a webpage. You can mess with its color, position, and size, to begin with, as well as any special effects you want the div to have, like animations or shadows, that are different from the rest of the page.

    If you want to create a sort of unique part of your web page and no other element like header, p (for paragraph), or section will do, then use a div.

    Then, if you go to the file called style.css and scroll to the bottom of the pafe, you'll find the special, specific styles for those div elements.

  3. Change display

    TL;DR:: HTML elements can be inline and wrap within a line or block and occupy their own lines (or sometimes mix both).

    In a webpage, the display attribute determines how each element 'stacks' or 'wraps' with other elements. Elements that are inline, like <strong> (for bold text), fit, or wrap, inside the lines of codes in which they appear. These inline elements appear horizontally, or side-by-side, without making new line breaks. They line up like blocks in a row.

    On the other hand, block elements occupy their own lines on the screen and trigger a new line break wherver they appear. These elements stack verically like blocks in a tower. For example, the <p> code, for paragraph, alwys starts a new line. Text inside the paragraph wraps, but the paragraph itself is a block element - a line break happens before and after it.

    Each HTML element has a defauly display setting of inline or block, but those settings can be changed through styling in .css files. Our div elements in index.html are set to be inline-block which lets them occupy the same amount of space they would normally occupy, but also to line up next to one another.

    Go to the style.css file and find the display element for each box div. Change the display value to inline for each block. What happens?

    Then change the display value to block for each bix div. What happens?

    Finally, change the display value for each block div back to inline-block for the rest of the activity.

  4. Change background-color and border color

    TL;DR: you can use color codes and names to change the color of elements on a webpage.

    There are plenty of ways to change the color of elements on a webpage using .css files. If you are in a 'TL;DR' mood, just mess around with color names and hexcodes today, and skip the last section on rgba colors.

    Websafe color names

    To begin, go to our style.css file and visit the div elements representing our boxes at the bottom of the page.

    Each box div has two colors, a background-color attribute and a color value for its border attribute.

    You can change colors three ways with HTML and CSS. First, you can just type 'websafe' color names, like 'red,' 'green,' or 'blue.' Here is a site from w3schools.com with color names (and hex codes, which we'll get to in a moment). Try typing some of these names into the CSS instructions for our boxes. When you delete the color codes that are already there for background-color and the last part of each border element, be sure to delete the #, as well.

    Hexcodes

    Next, let's play with hexcodes. Color hexcodes (from the Greek 'hex' for 'six') are six-character color codes that look like this: #000000.

    In a color hexcode, the first two characters are red values, the next two characters are green values, and the last two characters are blue values. Characters can be any number between 0 and 9, or any letter between 'a' and 'f.' The number 0 is the darkest value, and the letter 'f' is the brightest value, so #000000 displays as black, while #ffffff displays as white. Something like #ff0000 would be a bright red, while #550000 would be a darker red. If ever you have the same two values for each color in a hexcode, like in #ff0000 where both red values are 'f,' both blue values are 0, and both green values are 0, you can shortcut the code down to three characters like this: #f00. That is to say, #ff0000 equals #f00. Unless you're building a webpage for printing, it's good to use hexcodes.

    Use a site like that w3schools.com list of colors or a chart like this one to pick hexcodes and mess with the background-color and border color value for each box.

    RGBA values

    Finally, we can also use 'rgba' values for a color. The abbreviation 'rgba' stands for, 'red, green, blue, alpha,' where 'alpha' is a value for opacity. All rgba values are between 0 and 255, which represent the 256 steps (0 is the first step) a color can take in an 8-bit binary number. Circuitry runs on those 8-bit binary numbers as electricity defines how much of a circuit is on or off. Those 8-bit binary numbers control the color display and intensity of electronics like multi-color LEDs or color-printer heads, so when you use rgba values, you're essentially borrowing analog, physical color values created by passing current through part of a circuit that creates a physical response to the amount of voltage it receives. Hence, 'rgba' values are great to use on webpages that see frequent printing or that want to approximate the colors of, say, an LED connected to a microprocessor like the arduino.

    In our style.css file, you can use rgba values like this: rgba(0, 0, 256, 1). In an rgba code, the first number represents the red value, the second is the green, the third is the blue, and the fourth is opacity. Colors can go from 0 (darkest) to 255 (brightest), and alpha, or opacity, can go from 0 (transparent) to 1 (opaque) using decimal values, as well.

  6. Change position

    TL;DR: You can place an element like a div anywhere on a webpage using its stylesheet.

    As you take one more look at our boxes' div elements inside style.css, you can see that each has a position. The position of each div element works with its left and top attributes. For example, an element like one of our boxes that is positioned as relative with left and top values of 25px is positioned 25 pixels down from the top of the part of the page it's in, as well as 25 pixels to the left of the last part of the page that appears before it.

    There are several values for the position attribute. Here are three commonly used values:

    • Absolute - this kind of positioning anchors an element at a specific place, but still allows for page scrolling. Elements positioned with absolute depend on the entire page.
    • Fixed - this kind of positioning anchors an element at a specific place and anchors the whole page at the same time, preventing scrolling. Elements positioned with fixed depend on the entire page.
    • Relative - this kind of positioning places an element relative to the part of the page it's in or to the last element that appears before it. Elements positioned with relative depend on spatial relationships to other parts of the page rather than to only the entire page.

    Once you choose a value for position, you can place an element using top, bottom, left, or right. You can use either top or bottom and either left or right. You can also position something just from the top or bottom or just from gthe left or right.

    Dive back into style.css and see what you can 'break' by changing the value of position to absolute or fixed. You can also increase or decrease the value for top and left. Finally, you can also see what happens when you change top to bottom or left to right.
